## Part 1 — The findings (what we worked out)

### F1. The recursive implementation hides everything as witness
The outer circuit takes the K inner proofs **and their public inputs** as private
witness, verifies them in-circuit (`verify_honk_proof`), and re-runs the glue logic
against those now-trusted-but-private values. Only `(root, threshold)` are public —
the same surface as flat_merkle. So recursion's public surface = perfect/structural
hiding. (`tests/recursion_micro/exp2_k_segments/src/main.nr`.)

### F2. The external Python verifiers are a minimal, sound binding layer
`verify_hier.py` / `verify_hier_fs.py` run `bb verify` ×(K+1) then do **pure
Field-equality cross-checks** (no field arithmetic) binding the proofs to one
instance. `bb verify` alone only proves each proof internally consistent — nothing
binds K+1 independent proofs to the *same* instance. Justification: the checks are
deterministic, public, assumption-free integer comparisons on values `bb verify` has
already cryptographically bound; they expand the *trusted code* surface, not the
*cryptographic* trust base; they faithfully mirror the in-circuit asserts recursion
uses. This is conventional for aggregation/PCD/rollup verifiers.

### F3. The binding tax — one artifact, three symptoms
Decomposing into K independent segment-proofs forces you to bind them back into one
statement. That binding is **a single object with three coupled symptoms**, all of
which are the *shared public surface of independent proofs*:
1. **partition leakage** (the shared values are disclosed),
2. **O(K) verifier cost** (K+1 proofs → K× proof bytes + K× verify time),
3. **verifier-side bookkeeping** (the equality cross-checks).

They dissolve **together** under one move: fold the binding into a proof. Recursion
does this (the asserts become in-circuit; verifier returns to one `bb verify`).
Folding does it cheaply.

### F4. The pick-two triangle
At fixed privacy, three desiderata; each architecture gives exactly two:

| | Parallel + low per-prover mem (**P**) | O(1) verifier (**V**) | Low prover overhead (**C**) |
|---|:--:|:--:|:--:|
| flat (K=1) | ✗ | ✓ | ✓ |
| A / A++ / committed-* | ✓ | ✗ | ✓ |
| recursion | ✓ | ✓ | ✗ |
| **folding** (future) | ✓ | ✓ | ✓ (breaks the triangle) |

The O(K) verifier cost is **the price A/A++ pay to keep P and C together** — a
defining feature of the corner, not a defect.

### F5. The family is generated by two binding decisions
- **Where binding lives:** verifier-side (A, A++, B) / in-circuit (recursion) /
  deferred (folding).
- **What is bound:** plaintext (discloses — the leak; good when disclosure is the
  goal) / hiding commitment (committed-A/A++) / witness (recursion — the limit).

### F6. The commitment fix closes the leak without recursion
Bind on **hiding commitments** instead of plaintext: each sub exposes
`C_i = Commit(values; r_i)`, the glue takes the values+openings as witness and does
the partition/boundary/cost checks in-circuit, the verifier checks `sub_i.C_i ==
glue.C_i`. Public surface collapses to `(root, threshold, C_0..C_{K-1})`. The
bookkeeping becomes ZK automatically (comparing opaque blobs). Can be collapsed to one
digest per proof → O(K) single-field checks.

### F7. A is NOT dominated by A++ (the key correction)
Glue-to-glue, A++'s grand product beats A's sort (N=480 K=2: 6,987 gates / 31 MB vs
14,822 / 159 MB). **But A++ relocates that O(N) work into the K segments** (+~3.5% per
sub), so on totals A is *cheaper at every measured K*:

| Total gates (N=480) | K=2 | K=4 | K=8 |
|---|---|---|---|
| A | 769,926 | 775,622 | 787,014 |
| A++ | 788,533 | 794,401 | 806,137 |

Per-prover memory **ties at K=8** (~159 MB) for opposite reasons (A pinned by O(N)
glue floor; A++ by its heavier sub). A also has a **deterministic** partition check
(no Schwartz–Zippel error). So A wins at moderate K / total-gates / simplicity.

### F8. A++'s real justification is the recursion bridge + high-K corner
A++ earns its place via, in order of durability:
1. **Best recursion inner — O(1) public surface (9 fields, M-independent).** Keeps the
   recursive verifier segment-size-independent: single-segment outer = **704,363 gates
   flat at N=8 *and* N=480**. An A inner exposes M+4 fields → outer grows with N.
2. **High-K memory:** O(K) glue removes A's O(N) memory floor; A++ keeps falling at
   K≥16 where A plateaus.
3. **Distributed partition check:** O(M)/segment parallel + O(K) merge vs A's serial
   O(N) sort.
4. **O(K) standalone verifier surface** (washes out under the commitment fix).

So A++ is *the design you recurse on*, not a cheaper A. If recursion is cut from the
thesis, A++'s standalone case gets thin.

### F9. Commitment hiding is computational (with the efficient scheme)
- Poseidon/hash commitment → **computational** hiding (cheap, natural in Noir).
- Pedersen-style → **unconditional content** hiding, but costly in-circuit + only
  computational *binding*.
Even perfectly-hiding committed-* is **a notch weaker than recursion/flat**: it still
publishes K commitments (reveals K; hiding rests on the scheme), whereas
recursion/flat make the partition **structurally absent** (witness) — assumption-free.

### F10. "Perfect hiding" of flat/recursion is *structural*, not IT-ZK
It means the **public surface contains no route/partition information**, not that the
SNARK gives information-theoretic zero-knowledge (UltraHonk-ZK is computational/
statistical and is identical across all variants — not a discriminator).

## Part 2 — Reframed thesis spine

**Two structural results, complementary:**
- **Dualism (existing, Ch 8):** decomposition gives no algorithmic ZK speedup; its only
  payoff is parallelism / per-prover memory.
- **Binding tax (new):** the cost of binding K independent proofs back into one sound,
  private statement — one artifact, three symptoms (F3) — organizes the whole variant
  family via two decisions (F5).

**Frontier = pick-two triangle at fixed privacy (F4)**, with folding as the missing
corner. Privacy becomes a *held-constant condition* (commitment fix) or a *measured
axis* (privacy ladder), not the thing that makes variants incomparable.

**Progress-line (each step removes exactly one symptom):**
1. **flat-Merkle** — monolithic; perfect hiding, O(1) verifier, no parallelism.
2. **→ A** *(buy parallelism)* — binding tax appears: bookkeeping + plaintext partition
   leak. Cheapest total gates, deterministic.
3. **→ A++** *(shrink public surface + un-serialize glue)* — P_i + distributed check.
   NOT cheaper overall; a surface/critical-path optimization. Payoff deferred to step 5.
4. **→ committed-A/A++** *(equalize privacy)* — bind on hiding commitments; leak closes;
   only the O(K) verifier symptom remains.
5. **→ recursion** *(O(1) verifier + perfect hiding)* — bind in-circuit; all three
   symptoms collapse at once. Prover pays 704k×K. **Here A++'s O(1) surface pays off.**
6. **→ folding** *(drop the prover tax)* — defer verification; the open corner. Future.

## Part 3 — Per-implementation privacy classification

Common to all: route hidden by SNARK-ZK; ZK only hides beyond what the statement
implies.

| Implementation | Cost matrix | Partition | Partition privacy class |
|---|---|---|---|
| flat_full (4 perm variants) | disclosed (public) | N/A | — |
| flat_merkle_presence | committed (root) | N/A | — |
| **Variant A** | committed | node-sets + endpoints plaintext | **disclosed** (unconditional leak) |
| **Variant A++** | committed | endpoints plaintext; membership = unblinded `P_i` | **computational, oracle** (~C(N,M), confirms guesses) |
| **committed-A/A++ (hash)** | committed | blinded commitment | **computational** |
| **committed-A/A++ (Pedersen)** | committed | blinded commitment | **unconditional content** (K revealed, comp. binding) |
| **Recursion** (A- & A++-inner) | committed | absent (witness) | **structural / assumption-free** |
| Variant B (unimpl.) | sub-matrices disclosed | partition + sub-matrices plaintext | **disclosed** (most) |
| Folding (future) | committed | absent (witness) | **structural / assumption-free** |

**Privacy ladder (assumption-decreasing):**
B → A → A++ → committed(hash) → committed(Pedersen) → recursion/folding/flat.
Two distinct mechanisms: *commit to hide it* (rungs 4–5) vs *don't put it there at
all* (rung 6, assumption-free). A- and A++-inner recursion have **identical** final
privacy.
